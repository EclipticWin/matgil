# 26. 맛길 프론트엔드 리팩토링 지시서

## 작업 목적

현재 맛길(Matgil) 리액트 프론트엔드 프로젝트는 Claude Code 기반 바이브 코딩으로 빠르게 구현되면서 많은 페이지, 컴포넌트, 서비스, 유틸 파일이 생긴 상태다.

앞으로 기능을 계속 추가할 예정이므로, 지금 단계에서 전체 프론트엔드 코드를 점검하고 재사용 가능한 부분을 분리하여 유지보수하기 쉬운 구조로 정리한다.

이번 작업은 **기능 추가가 아니라 순수 리팩토링**이다.

## 프로젝트 전제

이 프로젝트는 개인 블로그 프로젝트가 아니다.

이 프로젝트는 맛길(Matgil) 프론트엔드 프로젝트다.

* 프론트엔드: Vite + React + JavaScript + Tailwind CSS
* 백엔드: Supabase
* VSCode 프로젝트 내부에는 프론트엔드 코드만 존재한다.
* Supabase 관련 백엔드 로직은 프로젝트 외부에 있다.
* 따라서 프론트엔드 내부 코드만 리팩토링 대상으로 본다.

1. 현재 기능 변경 금지

7. 현재 화면 배치 변경 금지
8. 현재 색상, 여백, 폰트 크기, 카드 모양, 버튼 모양 변경 금지
9. 기능 추가 금지

사용자가 화면에서 알아차릴 수 있는 변화가 있으면 안 된다.

## 이번 리팩토링에서 허용되는 작업

이번 작업에서 허용되는 것은 코드 내부 구조 정리뿐이다.

허용되는 작업은 다음과 같다.

* 중복 코드 제거
* 재사용 가능한 컴포넌트 분리
* 재사용 가능한 유틸 함수 분리
* 재사용 가능한 상수 분리
* 커스텀 훅 분리
* 너무 긴 페이지 컴포넌트 정리
* 역할이 섞인 컴포넌트 분리
* import 정리
* 사용하지 않는 코드 제거
* 파일명 개선
* 폴더 구조 정리
* 동일한 UI를 유지하는 범위 안에서 JSX 구조 정리

## 우선적으로 찾아야 할 리팩토링 후보

### 1. 공통 UI 컴포넌트

여러 페이지에서 반복되는 UI가 있다면 공통 컴포넌트로 분리한다.

예시:

* 버튼
* 카드
* 섹션 제목
* 페이지 헤더
* 빈 상태 화면
* 로딩 화면
* 에러 안내
* 배지
* 태그
* 모달
* 바텀시트
* 하단 네비게이션
* 모바일 프레임 레이아웃

단, 분리 후에도 기존 UI와 완전히 같아야 한다.

### 2. 기능별 컴포넌트

특정 기능 안에서만 쓰이는 컴포넌트는 해당 기능 폴더 아래로 정리한다.

예시:

```txt
features/
  auth/
  recommendation/
  courses/
  phrases/
  bookmark/
  popular/
  profile/
  navigation/
```

단, 현재 프로젝트 구조가 이미 다르게 정리되어 있다면 억지로 이 구조를 만들지 않는다. 기존 구조를 존중하면서 정리한다.

### 3. 상수 분리

여러 파일에서 반복되는 값은 상수로 분리한다.

예시:

* 라우트 경로
* 메뉴 이름
* 카테고리 목록
* 필터 옵션
* 기본 지역 목록
* 기본 문구
* 상태값
* 탭 목록

단, 실제 값은 변경하지 않는다.

### 4. 유틸 함수 분리

컴포넌트 내부에 반복되는 계산 로직이 있다면 유틸 함수로 분리한다.

예시:

* 거리 포맷
* 시간 포맷
* 날짜 포맷
* 텍스트 자르기
* 빈 값 처리
* 배열 정렬
* 필터링
* Supabase 응답 정리
* 북마크 여부 판별
* 코스 요약값 계산

단, 함수 분리 후 결과값이 기존과 달라지면 안 된다.

### 5. 커스텀 훅 분리

상태 관리와 부수효과가 복잡한 컴포넌트는 커스텀 훅으로 분리할 수 있다.

예시:

* 로그인 상태 확인
* 추천 데이터 로딩
* 북마크 목록 로딩
* 프로필 정보 로딩
* 모달 열림/닫힘 상태
* 바텀시트 상태
* TTS 재생 상태

단, 필요 없는 훅을 억지로 만들지 않는다. 컴포넌트가 너무 길거나 재사용성이 있을 때만 분리한다.

## 파일명 기준

파일명은 역할이 명확하게 드러나야 한다.

좋은 예:

```txt
RecommendationCard.jsx
CourseSummaryCard.jsx
PhraseCategoryTabs.jsx
BookmarkEmptyState.jsx
useBookmarks.js
useAuthUser.js
formatDistance.js
routeConstants.js
recommendationConstants.js
```

나쁜 예:

```txt
Card.jsx
List.jsx
Data.jsx
Temp.jsx
Test.jsx
Utils.jsx
Component.jsx
```

파일명을 바꿀 경우 import 경로를 모두 정확히 수정한다.

## import 정리 기준

* 사용하지 않는 import 제거
* 사용하지 않는 변수 제거
* 파일 이동 시 import 경로 전체 수정
* 너무 깊은 상대경로가 반복되면 구조 개선 검토
* circular dependency가 생기지 않게 주의
* 한 파일에서 너무 많은 역할을 import하고 있다면 컴포넌트 분리 검토



## 최종 지시

이번 작업은 맛길 프론트엔드 프로젝트의 유지보수성을 높이기 위한 리팩토링이다.

기능을 더 좋게 만들거나 디자인을 개선하는 작업이 아니다.

현재 사용자에게 보이는 화면과 기능은 그대로 유지한다.

코드 내부 구조만 더 읽기 쉽고, 재사용하기 쉽고, 앞으로 기능을 추가하기 쉬운 상태로 정리한다.
