# 00-project-brief.md

## 프로젝트명

맛길

## 서비스 설명

맛길은 외국인 관광객을 위한 서울 맛집 동선 추천 웹 애플리케이션이다.

사용자가 서울 내 지역 또는 랜드마크를 선택하면, 맛집 2곳과 카페 1곳으로 구성된 동선을 추천하고, 식당에서 자주 쓰는 한국어 표현을 TTS로 제공한다.

## 현재 작업 목표

현재 목표는 완성된 서비스 개발이 아니다.

이번 작업의 목표는 **Vite + React + Tailwind CSS 기반으로 맛길 MVP 화면 껍데기를 먼저 구현하는 것**이다.

실제 백엔드 연동, DB 연동, AI API 호출, 지도 기능은 구현하지 않는다.

mock 데이터와 mock 함수 기반으로 화면 흐름만 완성한다.

## MVP 핵심 흐름

1. 사용자가 홈 화면에 접속한다.
2. 서울 내 지역 또는 랜드마크를 선택한다.
3. 취향을 선택하거나 건너뛴다.
4. 맛집 2곳 + 카페 1곳 추천 결과를 생성한다.
5. 추천 결과를 지도 대신 카드 형태로 확인한다.
6. 식당에서 자주 쓰는 한국어 표현을 확인하고 TTS로 재생한다.

## MVP에서 검증할 핵심 가치

* 외국인 관광객이 서울 내 특정 지역의 맛집 동선을 빠르게 확인할 수 있는가
* 지도 없이도 카드형 추천 결과만으로 코스를 이해할 수 있는가
* 식당 이용 시 자주 쓰는 한국어 표현을 바로 확인하고 음성으로 재생할 수 있는가

## 기술 스택

프론트엔드:

* Vite
* React
* JavaScript
* Tailwind CSS

향후 백엔드:

* Supabase
* Supabase Edge Functions
* Solar API

## 중요한 기술 조건

프론트엔드에서는 TypeScript를 사용하지 않는다.

프론트는 JavaScript 기반 React로 구현한다.

단, 추후 백엔드는 Supabase Edge Functions를 사용할 예정이며, Edge Functions 쪽에서는 TypeScript를 사용할 수 있다는 점만 고려한다.

* pgvector 기반 의미 검색
